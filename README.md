# research_program_fcs

## Что это за репозиторий

Этот репозиторий посвящён экспериментам с **LoRA** для **Tiny LLM**: небольших GPT-like decoder-only transformer моделей, обучаемых и донастраиваемых на базе `nanoGPT` Андрея Карпатого. Практически это не самостоятельный training framework, а исследовательский репозиторий с конфигами экспериментов, вспомогательными скриптами и сводными таблицами результатов.

Эксперименты выполняются поверх модифицированного репозитория `gdifer12/nanoGPT`, который, в свою очередь, основан на оригинальном `karpathy/nanoGPT` Андрея Карпатого. Поэтому данный репозиторий стоит рассматривать как слой над training codebase: здесь хранятся постановка экспериментов, их конфигурации и агрегированные результаты.

## Содержание репозитория

- `fine-tune-lora-2/config/*` — основные конфиги эксперимента;
- `fine-tune-lora-2/res/*` — сводные таблицы результатов;
- остальное в корне — вспомогательные файлы для полного воспроизведения пайплайна.

## Мотивация

**LoRA** (*Low-Rank Adaptation*) — это параметр-эффективный способ fine-tuning, в котором базовые веса модели замораживаются, а обучаемое обновление веса представляется в виде низкорангового разложения. На практике это обычно означает, что вместо полного обновления матрицы весов `W` обучается лишь добавка `ΔW = BA`, где `A` и `B` — сравнительно маленькие матрицы, а их ранг `r` существенно меньше исходной размерности слоя. Тогда итоговый слой можно интерпретировать как `W + ΔW`, но число обучаемых параметров оказывается заметно меньше, чем при полном fine-tuning.

Это может быть полезно сразу по нескольким причинам. Во-первых, уменьшается число обучаемых параметров, а значит — требования к памяти оптимизатора и к хранению checkpoint'ов. Во-вторых, LoRA позволяет быстро сравнивать разные варианты адаптации одной и той же базовой модели без необходимости каждый раз обновлять все веса. В-третьих, для исследовательских сценариев это удобный способ проверить, какие части модели действительно критичны для адаптации под новую задачу. При этом для **tiny LLM** выигрыш не гарантирован: сама модель уже маленькая, поэтому особенно интересно отдельно измерить, насколько снижение числа обучаемых параметров приводит к потере качества.

## Цели

Цель данного этапа — исследовать **потери качества в зависимости от применения LoRA для tiny LLM**. То есть понять, насколько сильно меняются метрики модели при разных схемах подключения LoRA.

## Эксперименты

Эксперименты проходят по следующему сценарию: 
1. Обучена и зафиксирована `base-model`. Она умеет только генерировать тексты.  
2. Она дообучается с различными надстройками чтобы соответствовать instruct-режиму.

### Датасеты

Для базовой модели был выбран датасет [TinyStories](https://huggingface.co/datasets/roneneldan/TinyStories). Он был выбран как компактный и уже хорошо известный датасет для исследований tiny LLM: позволяет быстро запускать и сравнивать большое число экспериментов при ограниченном вычислительном бюджете, сохраняя при этом осмысленную задачу autoregressive language modeling.

Для `fine-tuning` был выбран соответственно датасет [TinyStoriesInstruct](https://huggingface.co/datasets/roneneldan/TinyStoriesInstruct)

### Baseline

Полная конфигурация приведена в файле `fine-tune-lora-2/baseline/congig.py`.

### Experiments

Проведённые запуски из блока `fine-tune-lora-2`:

| ID | Описание |
|---|---|
| `F_1` | LoRA отключена, используется полный fine-tuning |
| `F_2` | LoRA применяется ко всем attention-блокам во всех слоях |
| `F_3` | LoRA применяется к attention-блокам в верхней половине слоёв |
| `F_4` | LoRA применяется в каждом втором слое |
| `F_5` | LoRA применяется к token embeddings (`wte`) |
| `F_6` | LoRA применяется к `wte` и каждому второму слою |

## Сводные таблицы результатов

Ниже приведена краткая сводка на основе файла `research_program_fcs/fine-tune-lora-2/res/summary_ftl_significant.csv`.

| Эксперимент | Доля обучаемых параметров | `best_val_loss_mean` | `final_val_loss_mean` |
|---|---:|---:|---:|
| `F_1` | 100.00% | 1.13004 | 1.13134 |
| `F_2` | 80.36%  | 1.14436 | 1.14516 |
| `F_3` | 90.16%  | 1.13860 | 1.13968 |
| `F_4` | 90.16%  | 1.13738 | 1.13858 |
| `F_5` | 59.87%  | 1.15140 | 1.15158 |
| `F_6` | 50.19%  | 1.17032 | 1.17042 |

По этой сводной таблице видно, что в данном наборе экспериментов наилучшее качество показывает `F_1`, то есть **полный fine-tuning без LoRA**. Среди LoRA-вариантов лучшим по `best_val_loss_mean` оказывается `F_4`, где LoRA применяется в каждом втором слое. При этом `F_6` даёт наименьшую долю обучаемых параметров, но и наиболее заметную просадку по качеству.
